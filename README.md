v1.8.5 — AI-driven 4-koma manga voiceover and video generation tool using Dual API Engine (Gemini & OpenAI) / Dual API Engine (Gemini & OpenAI) を使用したAI駆動の4コマ漫画フルボイス動画自動生成ツール

## 🚀 Overview / 概要
AI Voice Comic Maker is a fully automated tool that generates "dynamic, fully-voiced vertical short videos" from text or a single 4-koma manga image, allowing immediate compilation and export. It automatically detects the language (Japanese/English) and routes voice generation through VOICEVOX, Supertonic 3, or Edge-TTS depending on the selected local engine and script language.
AI Voice Comic Makerは、テキストや1枚の4コマ漫画画像から「ダイナミックなフルボイス縦型ショート動画」を生成し、一瞬でコンパイル・エクスポートできる完全自動化ツールです。漫画の言語（日本語・英語）とユーザーのTTS選択に応じて、VOICEVOX、Supertonic 3、Edge-TTSを使い分けて音声化します。

It provides seamless video assets for platforms like TikTok and YouTube Shorts, specifically optimized to read formats generated by Super FURU AI 4-koma System, supporting highly immersive comic expressions.
Super FURU AI 4-koma System などの漫画制作システムによって生成されたフォーマットに特化し、TikTokなどのプラットフォーム向けの動画素材として没入感のある表現をサポートします。

## ⚠️ Local Execution Only / ローカル実行専用
> **This application does NOT work as a hosted web demo.** It requires a local Node.js backend server plus at least one local Japanese TTS engine: VOICEVOX on `localhost:50021` or Supertonic 3 on `localhost:7789`. Unlike browser-only apps (e.g., Super FURU AI 4-koma System), this system performs server-side video rendering via Remotion (headless Chromium) and communicates with local TTS services, which are impossible from a static hosting service like GitHub Pages.

> **本アプリはWebデモとしては動作しません。** ローカルのNode.jsバックエンドサーバーに加えて、日本語TTSとして `localhost:50021` のVOICEVOX、または `localhost:7789` のSupertonic 3 のどちらかが必要です。他のブラウザ完結型アプリ（Super FURU AI 4-koma System等）とは異なり、本システムはRemotionによるサーバーサイド動画レンダリング（ヘッドレスChromium）とローカルTTSサービスとの通信を行うため、GitHub Pagesのような静的ホスティングでは動作できません。

## ✨ Features / 機能

### 🧠 Dual API Architecture (Gemini & OpenAI 両対応)
- **Automatic Engine Detection / 自動エンジン認識**: Enter either a Gemini API Key (`AIza...`) or an OpenAI API Key (`sk-...`), and the backend automatically routes the request to the optimal vision model. / ユーザーが入力したAPIキーの形式から、バックエンドが自動的に「Gemini」か「OpenAI (gpt-4.1 fallback chain)」かを認識し、最適なルーティングを行います。手動のトグルスイッチは不要です。
- **Accuracy & Contextual Fallback / 精度と文脈補完**: Gemini is recommended as the primary engine due to its superior Japanese OCR capabilities. However, when using OpenAI, the system applies a "Contextual OCR Verification" prompt to self-correct kanji misreadings based on conversational context. / 日本語OCRの精度において圧倒的に優れる **Gemini** の使用を強く推奨します。OpenAIを使用する場合は、誤読（ルビの誤認や濁点の欠落など）を軽減するため、AI自身に前後の文脈からテキストを再考証させる強力な自己検証プロンプトが実行されます。

## 🏗️ Unique Architecture Highlights / 固有アーキテクチャの要点
This system is not merely a video editor. It is an automated orchestration engine designed to convert static visual art into engaging auditory and spatial short-form content.  
本システムは単なる動画編集ツールではありません。静止画の視覚芸術を、魅力的な音声と空間的演出を伴うショート動画コンテンツへ変換するための自動化オーケストレーションエンジンです。

- **Contextual Reading & Tracking (文脈的読解とキャラクター追従)**: The AI does not merely process the manga as raw image data; it "reads" the narrative context. Even if a character's art style drastically changes across panels, the Vision AI understands their role and narrative continuity. It assigns a consistent speaker identity based on context rather than pixel-level similarity, ensuring the assigned voice remains perfectly stable throughout the comic. / AIは漫画を単なる画像データとして処理するのではなく、ストーリーの「文脈」として読解します。コマ間でキャラクターの絵柄が劇画調からちびキャラへと急激に変化した場合でも、文脈や役割から「同一人物」であると正確に追従し、割り当てられた声優（音声ID）が途中で変わってしまう事故を防ぎます。
- **2-Pass AI Dialogue Auto-Correction (2-Pass AI テキスト自動校正)**: The distorted fonts generated by AI manga generators often trigger OCR misreadings (e.g., parsing "活躍" as "洋題" or "躍"). Standard 1-Pass OCR prompts fail to resolve this due to visual noise. We implemented a 2nd-pass contextual pipeline: after extraction, the text payload is sent to the LLM (Gemini/OpenAI) along with the comic title. Purged of visual data, the AI successfully self-corrects spelling and grammar based on narrative flow, aligning both subtitles (`text`) and voice output (`pronunciation`). / 画像生成AIで作られた漫画の文字は歪みや滲みが強く、1パスのVision OCRでは「活躍」が「洋題」や「躍」と誤認識されるケースが多発していました。本システムでは、1次OCR完了直後に画像を除外したテキスト情報（タイトル＋全セリフ）のみを再度AIに送信する **2-Pass校正パイプライン** を確立。AIが文脈のみに集中することで、「活躍してた」等の正しい表現へ高精度に自動修復し、字幕（`text`）と発音（`pronunciation`）を同期して補正します。
- **Unbiased Physical Reading Order Sort (配列順序バイアスの解除と物理ソート)**: Forcing the AI to output dialogues in a specific reading order causes the model to bend physical coordinate data (`bubblePosition`) to justify its output sequence. We removed all reading order constraints from the AI prompt, allowing unordered dialogue outputs. The backend then programmatically sorts dialogue from right-to-left before generating video, completely eliminating layout desync (camera-audio mismatch) bugs. / プロンプトでAIに「右から左に読め」と読む順を強制すると、AIが読む順（セリフ順）を勘違いした際に、物理配置（`bubblePosition`）のデータを捻じ曲げて出力してしまい、カメラと音声がテレコになるバグが生じていました。AIへの順序制約を完全に撤廃して物理的な位置分類に専念させ、バックエンドで一律に右➔左へ物理ソートを行うことで、このバグを根本解決しました。
- **Multi-Panel Dialogue Deduplication (コマ跨ぎ重複セリフの自動除去 - Dedup Engine)**: If the Vision AI erroneously repeats the same speaker/dialogue combination across adjacent panels, the system detects these duplicates and automatically keeps the latter occurrence (where the bubble is usually captured more clearly) while purging the redundant entries. / GeminiなどのVision OCRが隣接するコマで同じセリフを重複して検出してしまった場合、システムが自動的に重複を検知し、吹き出しがより鮮明に描かれているはずの「後方コマ」のデータを正本として残し、重複した前方コマの不要データを自動除去（デダップ）します。
- **Super FURU Format Optimization (専用フォーマット最適化)**: Instead of randomly slicing panels, it specifically removes top/bottom headers and footers associated with the Super FURU AI 4-koma System system, slicing the manga into 4 perfect horizontal panels to minimize structural collapse. / ランダムにコマを切り出すのではなく、Super FURU AI 4-koma System システム特有の上下のヘッダー・フッターを正確に除外し、4つの横長コマとして完璧にスライスすることで、構造の破綻を最小限に抑えます。
- **Autonomous English Narration & Edge-TTS Pipeline (自律的な英語ナレーションと Edge-TTS 統合パイプライン)**: When the system detects that the comic script is written in English (via early linguistic check), it automatically bypasses the VOICEVOX pipeline (which only supports Japanese) and routes both panel dialogues and the title call directly to the Microsoft Edge-TTS engine. To ensure fluent and correct English speech, the system dynamically suspends all Japanese Katakana dictionary processing and passes raw English text directly to the designated voices (e.g. `en-US`, `en-GB`, `en-AU` Neural voices), eliminating silent rendering failures and phonetic errors. / 英語原稿判定時（自動検出）には、日本語専用の VOICEVOX 音声合成パイプラインを完全に迂回し、Microsoft Edge-TTS エンジンにダイレクト接続します。不自然なカタカナ変換処理（発音辞書）を自動的にスキップし、生の英文をそのまま en-US, en-GB, en-AU などの高品質な Neural 音声で再生することで、無音バグを完全に排除し、極めて流暢でネイティブな英語読み上げを実現します。
- **Supertonic 3 Local Fallback Pipeline (Supertonic 3 ローカルフォールバック統合)**: The app can now use Supertonic 3 as a fully local TTS engine through a Python HTTP server on `localhost:7789`. The UI preserves the user's selected TTS engine through both analysis and generation, the backend writes `ttsEngine` into Remotion `scriptData`, and the rendered credit switches to `Sound: Supertonic 3` when used. / Python HTTPサーバーとして動作する `localhost:7789` のSupertonic 3を、完全ローカルTTSエンジンとして利用できます。UIで選択したTTSエンジンは解析と生成の両方に引き継がれ、バックエンドはRemotion用 `scriptData` に `ttsEngine` を書き込み、使用時のクレジットも `Sound: Supertonic 3` に切り替わります。
- **Final MP4 Loudness Mastering (最終MP4音量マスタリング)**: After rendering, the server remasters the audio stream to a practical loudness target using ffmpeg, preventing Supertonic-generated voices and BGM from becoming too quiet in Windows video players or browser playback. / レンダリング後にffmpegで音声ストリームを実用音量へマスタリングし、Supertonic生成音声やBGMがWindows動画プレーヤーやブラウザ再生で小さすぎる問題を防ぎます。
- **Stateful Voice Mapping (状態保持ボイスマッピング)**: Implements a tracking memory algorithm to prevent the same voice ID from being assigned to different characters in a single session. / 単一セッション内で異なるキャラクターに同じ音声IDが割り当てられるのを防ぎます。
- **Alphanumeric Pronunciation Standardization (英数字・ガジェット・AI用語の発音標準化)**: The system features an extensive dictionary of over 100+ preset terms, resolving the open-source text-to-speech engine's native struggle with western alphanumeric brands. Instead of reading "Switch2" as "Switch-ni" (interpreting 2 as the Japanese word "ni"), or splitting "Webtoon" as "Web-TOON", the backend programmatically standardizes them to natural Japanese kana pronunciations (e.g. "スイッチツー", "ウェブトーン"). This vocabulary spans gaming hardware (Switch2, PS5 Pro), smart devices (iPhone16, Quest 3), OS (Windows 11), AI engines (GPT-4o, Claude 3.5), network protocols (5G, Wi-Fi 7), and scientific concepts (CO2, PM2.5). / VOICEVOXなどの音声合成エンジンが苦手とする英数字交じりの固有名詞や技術用語を、自然な日本語発音に標準化する辞書システムを実装。例えば「Switch2」が「スイッチに」と誤読されたり、「Webtoon」が「ウェブ・トゥーン（変なイントネーション）」と誤認識される問題を回避するため、100語以上のプリセット（Switch2, PS5 Pro, iPhone16, Windows 11, GPT-4o, Claude 3.5, 5G, Wi-Fi 7, CO2, PM2.5等）を登録。AI漫画で頻出するあらゆるテクノロジー・サブカル用語を滑らかに発音させます。
- **AI Dynamic Pronunciation Lookup Pipeline (AI動的発音推定パイプライン)**: For unknown English words or proper nouns not registered in the preset dictionary, the system automatically detects them (using a 3+ character regex filter) and queries the AI (Gemini or OpenAI) in real-time to obtain the standard Japanese Katakana pronunciation. To prevent redundant API calls, results are cached per session. This ensures that even obscure technical terms, new product names, or character names are read naturally in Japanese without silent failures or phonetic breakdown. / 静的辞書に登録されていない未知の英単語や固有名詞に対して、バックエンドが自動的に検出し（3文字以上の英単語が対象）、Gemini や OpenAI を使用して動的に一般的なカタカナ読みをリアルタイムで推定・置換する機能を実装。同一セッション内での重複APIコールを避けるため、推定結果はセッション単位でキャッシュされます。これにより、辞書にない最新のIT用語やキャラクターの固有名詞でも、不自然な誤読を極限まで減らして滑らかに読み上げることができます。

## 🧠 Zenith Protocol（AIモデル自動切替 / Auto AI Model Fallback）
Following the philosophy of Super FURU AI 4-koma System, this system features a robust fallback mechanism (Zenith Protocol) that automatically switches to optimal alternative models upon API errors, rate limits, or safety filter blocks.
Super FURU AI 4-koma System の思想を踏襲し、APIエラー時や制限到達時、あるいは安全フィルタでのブロック時に自動的に最適な別モデルへフォールバックする仕組み（Zenith Protocol）を搭載しています。

All model lists are centralized in a single `AI_MODELS` configuration block at the top of `server.js`. To swap models, edit only this one block — all downstream references update automatically.  
全モデルリストは `server.js` 先頭の `AI_MODELS` 設定ブロックに集約されており、モデル入替時はこの1ブロックだけ変更すれば全箇所に反映されます。
