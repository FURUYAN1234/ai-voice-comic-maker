## [v1.4.6] (2026-05-24) - 英数字・ガジェット・AI用語の発音補正辞書の大幅拡充

### 変更内容
1. **発音補正辞書の大幅拡充 (`server.js`)**
   - ゲーム・エンタメ: `Switch2 / Switch 2`, `PS5 Pro`, `PS VR2`, `Quest 3`, `1UP`, `2ch`, `5ch`, `LEVEL5` 等
   - スマホ・タブレット: `iPhone 16 〜 13`, `iPad Pro / Air / mini`, `Apple Watch`
   - OS・通信: `Windows 11 〜 7`, `Wi-Fi 7 / 6 / 6E`, `5G`, `4G`, `USB-C`
   - AIモデル: `GPT-4o`, `Claude 3.5`, `Gemini 1.5`, `Llama 3`, `SD3`, `SDXL`
   - ミリタリー・サイエンス: `F-15`, `AK-47`, `CO2`, `PM2.5`, `A4` / `B5` 用紙サイズ等
2. **バージョン同期 (v1.4.6)**

### 検証結果

## [v1.4.5] (2026-05-23) - 2-Pass AIテキスト自動校正と物理配置ソート

### 目的
画像生成AIの漫画特有のフォントの歪み・滲みによるOCR誤読を自動修復する「2-Pass AIテキスト自動校正」を導入し、さらに配列順序バイアスを排除して物理的な吹き出し位置座標（bubblePosition）にのみ基づいた物理ソートをバックエンドで実行することで、カメラと音声のテレコバグを根本解決する。また、不要な重複セリフを除去する「Dedup Engine」を導入する。

### 変更内容
1. **2-Pass AI テキスト自動校正 (`server.js`)**
   - 1次パース完了後に、抽出されたタイトル＋全セリフのテキストJSONのみを再度AI（OpenAI/Gemini）に送信。
   - 純粋な文脈とストーリーの流れに集中させ、「躍してた」➔「活躍してた」などの誤読を自動修復する校正パスを確立。字幕（`text`）と発音（`pronunciation`）を同期して補正する。
2. **物理配置ソートへの完全移行とAI順序制約の撤廃 (`server.js`)**
   - プロンプトから「右から順に出力せよ」という順序制約を完全に排除し、AIには物理的な吹き出し位置の分類（`left`, `center`, `right`）に専念させる。
   - バックエンド側で一律に `bubblePosition` に従って「右 ➔ 中央 ➔ 左」に配列ソートを行うように修正。これにより読む順の齟齬による左右座標の歪曲バグ（テレコバグ）を解消。
3. **重複セリフ自動除去エンジン (Dedup Engine) の実装 (`server.js`)**
   - 同一 speaker + text が隣接するコマに現れた場合、後方コマのデータを正本として残し、重複した前方コマの不要データを自動除去（デダップ）。
4. **バージョン同期**
   - バージョンを `1.4.5` に引き上げ、`package.json`, `src/App.jsx`, `index.html`, `README.md`, `docs/walkthrough.md` を同期。

### 検証結果
- `npm run dev` でローカル開発サーバーが正常起動。
- 添付された4コマ漫画画像「WEBTOON解散の衝撃」のAI解析・動画生成テストにおいて：
  - OpenAI OCRで「活躍」が「躍」と誤読された部分が、2-Pass校正により「活躍してた」へと正しく自動修復されることを確認。
  - 3コマ目の左右反転バグ（テレコバグ）が完全に解消され、音声とカメラワークがキャラクターの吹き出し配置と一致することを確認。
  - コマを跨ぐ重複セリフが「Dedup Engine」により自動で検知・除去されることを確認。

## [v1.4.4] (2026-05-23) - 表示（text）と発音（pronunciation）の分離

### 目的
英語やアルファベット（例: 「WEBTOON」や「AI」等）が含まれるセリフにおいて、字幕表示用のテキストは英語のままとし、VOICEVOXの音声合成には自動で日本語の読み（カタカナ）を適用する二重構造（表示と発音の分離）を実装・検証する。

### 変更内容
1. **VOICEVOX音声合成ロジックの改善 (`server.js`)**
   - 音声合成のリクエスト送信時、AIによって生成された `pronunciation`（発音の読み）フィールドが存在すればそれを優先的に使用し、存在しない場合のみ従来の `text`（字幕表示用）へフォールバックするように変更。
   - `applyPronunciationDict(d.pronunciation || d.text, sessionId)` として、発音辞書も併せて適用。
2. **タイムライン構築データへの追加 (`server.js`)**
   - 生成される `scriptData.json` の各セリフオブジェクトに `pronunciation` フィールドを追加。
3. **進行ログの強化 (`server.js`)**
   - VOICEVOXの音声合成を実行する際、字幕テキストと同時にAIが生成した発音テキスト（例: `発音: "ウェブトーン"`）をターミナルログおよびコンソールにリアルタイムで出力し、検証可能に改善。
4. **バージョン管理同期**
   - バージョンを `v1.4.4` にインクリメントし、`package.json`, `src/App.jsx`, `index.html`, `README.md`, `server.js` を同期。

### 検証結果
- `npm run dev` による開発環境の正常起動を確認。
- 音声合成時、ログに `発音: "ウェブトーン"` などが正しく表示され、VOICEVOXがカタカナ音を読み込んで発声することを確認予定。
